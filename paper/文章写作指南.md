## 一、研究概述

### 研究背景
- **99mTc-EC 肾动态显像**：评估肾流出道梗阻的一线影像学方法
- **传统流程**：医生手工勾画 ROI → 提取 TAC 曲线 → 视觉+定量分析 → 诊断
- **痛点**：手工 ROI 耗时、存在观察者间变异、依赖经验

### 核心创新点
1. **UNet+LTAE 自动肾脏 ROI 分割**：替代医生手动标注，实现全自动流程
2. **两阶段 VLM 诊断流程**：Stage 1 提取结构化证据 → Stage 2 综合诊断，可解释性高
3. **三项正交消融实验**：系统量化 TAC、图像特征、自动 ROI 各组件贡献

### 目标期刊
- **European Journal of Nuclear Medicine and Molecular Imaging (EJNMMI)**
- 英文 Original Article
- 结构化摘要 ≤300 词，正文 ~3000–4000 词

## 二、数据流

```
130帧原始动态显像 (128×128, 2秒/帧)
        │
        ▼
26帧融合图像 (标准协议: 帧1-30均值 → 每4帧均值 → 帧128-130均值)
        │
        ▼
UNet+LTAE 自动分割肾脏 ROI (输入: 26通道融合图, 输出: 二值掩码)
        │
        ├──→ 左右肾分离 (连通域分析 + 质心x坐标)
        │        │
        │        ├──→ 肾脏裁剪 (11帧, 512×512, contrast 2.2, padding 0.4) → VLM Stage1
        │        │
        │        └──→ TAC提取 (130帧, 255反转, 均值, 归一化, 平滑) → TAC曲线图 → VLM Stage1
        │
        ▼
Stage 1: VLM (GPT-5.5, temperature 0.0) 提取11项结构化证据
        │
        ▼
Stage 2: LLM (同模型) 综合证据 → 四分类诊断
        │
        ▼
输出: 正常肾脏 / 功能性梗阻 / 机械性梗阻 / 混合性梗阻 + 置信度 + 推理
```

## 四、UNet+LTAE 模型

### 网络架构
```
输入: 26通道融合图像 (H, W, 26)
    │
    ▼
LTAE_Block (通道注意力)
    │
    ▼
Encoder: DoubleConv × 4 (64→128→256→512→1024)
    │  skip connections
    ▼
Decoder: UpConv + DoubleConv × 4
    │
    ▼
1×1 Conv → Sigmoid → 输出: 二值掩码 (H, W, 1)
```

### LTAE_Block 细节
```
AdaptiveAvgPool2d(1) → Flatten → FC(通道, 通道/4) → ReLU → FC(通道/4, 通道) → Sigmoid
→ 残差: output = input × attention_weight + input
```

### 训练配置
| 参数 | 值 |
|------|-----|
| 训练数据 | data3/ 独立队列，102 样本 |
| 分割 | 75/25（76 训练，26 验证），seed 42 |
| 输入尺寸 | 256×256 |
| Epochs | 200 |
| Batch size | 2 |
| 优化器 | Adam, lr=1e-4 |
| 损失函数 | 0.5×BCEWithLogitsLoss + 0.5×DiceLoss |
| LR调度 | ReduceLROnPlateau (factor=0.5, patience=5) |
| 数据增强 | RandomRotate15°, HorizontalFlip, VerticalFlip, ElasticTransform |
| 最佳模型 | 按验证集 Dice 选择 → best_model.pth (epoch 200) |

### 后处理
- 二值化: sigmoid(输出) > 0.5
- 连通域分析: 单域 → 质心x坐标分割左右；多域 → 取最大两域按x坐标分配左右

## 五、两阶段 VLM 诊断流程

### Stage 1: 结构化证据提取
- **输入**: 11帧肾脏裁剪图像 (512×512) + TAC 曲线图
- **模型**: GPT-5.5, temperature 0.0
- **输出**: 11 个结构化字段（受控词汇表）
- **11 字段清单**:

| 字段 | 受控词汇 |
|------|----------|
| 显影及时性 (early_visualization) | 及时 / 轻度延迟 / 严重延迟 |
| 达峰时间 (peak_timing) | 早期达峰 / 中期达峰 / 晚期达峰 |
| 利尿前排空趋势 (pre_diuretic_trend) | 已开始自然下降 / 基本平台或轻度滞留 / 持续上升或进行性积聚 |
| 利尿后下降 (post_drop_presence) | 明确连续下降 / 无明确下降或继续上升 |
| 利尿后排空充分性 (post_emptying_sufficiency) | 排空较充分 / 排空不充分 / 基本未排空 |
| 集合系统变化 (collecting_system_change) | 范围明显缩小 / 范围部分缩小 / 范围无明显缩小 |
| 利尿后响应 (post_diuretic_response) | 明显持续改善 / 有一定改善但不充分 / 几乎无明显改善 |
| 晚期滞留 (late_phase_retention) | 低 / 中 / 高 |
| 晚期滞留 vs 利尿时相 (late_retention_relative_to_diuretic) | 明显低于 / 略低于 / 接近或高于利尿时相 |
| 整体动态模式 (overall_dynamic_pattern) | 排空协调 / 排空延迟且后期改善不充分 / 持续滞留无实质改善 |
| 推理聚焦点 (reasoning_focus) | 自由文本 |

- **验证**: 输出 JSON 逐字段核对受控词汇表，未识别值替换为 "uncertain"

### Stage 2: 诊断综合
- **输入**: Stage 1 输出（双肾各 11 字段）
- **模型**: 同 GPT-5.5, temperature 0.0
- **输出**:
  - 每肾 final_label: 正常肾脏 / 功能性梗阻 / 机械性梗阻 / 混合性梗阻
  - 每肾 confidence: 低 / 中 / 高
  - 每肾 reasoning: 简短推理
  - overall_confidence: 整体置信度
  - image_tac_consistency: 图像TAC一致性

### TAC 曲线处理
- 255 反转（原始图像反色，高计数显暗）
- 掩码内像素均值
- 归一化至 [0,1]
- 5帧移动平均平滑
- 可视化并保存为图像输入 VLM

### 8.5 关键发现总结

1. **UNet+LTAE vs 医生ROI**: Δ=1.92%, p=0.625 — **无显著差异**
   - 各类别F1基本持平（正常0.920 vs 0.920, 功能0.698 vs 0.667, 混合0.545 vs 0.514, 机械0.889 vs 0.864）
2. **TAC是核心依据**: 移除TAC后准确率暴跌38.46% (80.77→42.31%, p<0.001)
3. **图像提供互补信息**: 仅TAC vs 完整流程有显著差异 (67.31 vs 80.77%, p=0.0013)
4. **机械性梗阻最好** (Sens 100%, F1 0.889), **混合性梗阻最难** (Sens 56%, F1 0.546)
5. **两阶段流程一致性**: 医生ROI和UNet+LTAE的错误模式高度一致

#### Discussion (已写)

**5.1 三个主要发现**
1. 自动ROI与手动ROI无显著差异（p=0.625）
2. TAC是核心依据（移除后准确率减半）

**5.2 各类别难度梯度分析**
- 机械性/正常：表型独特→高准确率
- 功能性：与正常和机械性重叠→中度
- 混合性：病因谱连续→最难

**5.3 局限性**
- 样本量小（52例），宽置信区间
- UNet+LTAE仅用26帧训练，可能丢失细粒度时间信息
- 仅用GPT-5.5，未跨模型泛化验证
- 8例混合性病例排除→选择偏倚
- 回顾性设计

**5.4 未来方向**
- 直接在评估队列上训练分割网络
- 端到端联合优化
- 分割级不确定性量化 → 选择性人工复核
- 前瞻性多中心验证

#### Conclusion (已写)
一句话总结：自动ROI可替代手动标注，TAC主导诊断，图像补充价值，两阶段流程临床可行性好。
