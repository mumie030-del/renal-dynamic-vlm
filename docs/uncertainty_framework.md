# 不确定性驱动三阶段诊断框架（Uncertainty-Driven Three-Stage Framework）

## 核心哲学
**图像优先，TAC按需引入**
- Stage 1：单次图像判读 + 自我不确定性评估
- Stage 2：量化不确定性（基于语义指标，非随机噪声）
- Stage 3：仅当不确定性高时，才引入TAC仲裁

## 📐 不确定性量化公式详解

### 完整公式

```
uncertainty_score = Σ(权重_i × 惩罚_i)

其中：
  = w_conf × conf_penalty
    + w_cons × cons_penalty
    + w_mixed × mixed_penalty
    + w_qual × quality_penalty

归一化：score_norm = min(score / max_possible, 1.0)
```

### 各分量计算

#### 1. 置信度惩罚 (conf_penalty)
```
right_conf_penalty = {high:0, medium:0.3, low:0.6}[right.confidence]
left_conf_penalty  = {high:0, medium:0.3, low:0.6}[left.confidence]

avg_conf = (right_conf_penalty + left_conf_penalty) / 2
```

**临床意义**：模型自己都不确定 → 需要专家会诊

#### 2. 证据一致性惩罚 (cons_penalty)
```
right_cons_penalty = {high:0, medium:0.2, low:0.5}[right.evidence_consistency]
left_cons_penalty  = {high:0, medium:0.2, low:0.5}[left.evidence_consistency]

avg_cons = (right_cons_penalty + left_cons_penalty) / 2
```

**临床意义**：证据自相矛盾 → 可能存在灰区或伪影

#### 3. 混合型惩罚 (mixed_penalty)
```
mixed_penalty = 0
if right.type == "混合性梗阻": mixed_penalty += 0.3
if left.type  == "混合性梗阻": mixed_penalty += 0.3
```

**临床意义**：混合型本身就是不确定的中间状态

#### 4. 图像质量惩罚 (quality_penalty)
```
quality_penalty = 0
for side in [right, left]:
  n_factors = len(side.uncertainty_factors)
  quality_penalty += min(n_factors * 0.15, 0.3)  # 每侧上限0.3
```

**临床意义**：模型自我报告的技术问题（伪影、模糊等）

### 归一化与阈值

```
max_possible_score = 0.6 + 0.5 + 0.3 + 0.6 = 2.0
uncertainty_score = min(score / 2.0, 1.0)

阈值选择：
  - τ = 0.4（推荐）：平衡敏感性与特异性
  - τ = 0.3（更激进）：更多病例触发TAC
  - τ = 0.5（更保守）：仅最模糊病例触发TAC
```

## 🎨 Prompt设计哲学

### Stage 1 Prompt设计要点

**核心目标**：让模型**同时给出判断和元认知**

**关键字段**：
1. `evidence_consistency`：强制模型反思证据是否自洽
2. `uncertainty_factors`：迫使模型识别技术问题
3. `confidence`：基于以上两点的综合置信度

**设计技巧**：
- 明确要求"自我评估不确定性"
- 举例说明什么算"uncertainty_factors"
- 强调"证据一致性"的重要性

### Stage 3 Prompt设计要点

**核心目标**：解决Stage 2识别的具体不确定性

**信息注入**：
- 明确告知"为什么触发"（uncertainty_score + reasons）
- 提供原判读的完整证据链
- 强调"请解决这些具体矛盾"

**设计技巧**：
- 不要泛泛而谈"请判断"
- 要具体："右肾证据矛盾：A vs B，请解决"
- 要求输出"uncertainty_resolved"字段

## 📈 阈值选择与调优策略

### 初始阈值：τ = 0.4

**理由**：
- 平衡点：约30-40%病例触发TAC
- 保留大部分明确病例（score<0.4）的纯图诊断
- 覆盖大多数混合型+低置信+质量问题

### 调优方法

```
1. 在验证集上扫描 τ ∈ [0.2, 0.6]
2. 评估指标：
   - 准确性（与金标准对比）
   - TAC使用率（成本）
   - 触发病例的改善率（TAC后是否正确）
3. 选择使F1最优的τ
```

### 预期触发率

| uncertainty_score | 占比（预估） | 处理方式 |
|------------------|-------------|---------|
| 0.0 - 0.3        | ~50%        | 直接输出 |
| 0.3 - 0.5        | ~30%        | 触发TAC  |
| 0.5 - 1.0        | ~20%        | 触发TAC  |

**总计**：约50%病例无需TAC，50%需要TAC仲裁

## 🏆 相比旧版的核心优势

| 维度 | 旧版（两次图不一致触发） | 新版（不确定性驱动） |
|------|----------------------|-------------------|
| **触发信号** | 外部（两次结果差异） | 内部（模型自我评估） |
| **抗随机性** | ❌ 易受temperature影响 | ✅ 单次稳定输出 |
| **可解释性** | ❓ "为什么触发？" → "因为两次不同" | ✅ "为什么触发？" → "因为证据矛盾+低置信" |
| **审稿友好** | ❌ 可被质疑"两次都不稳怎么办" | ✅ 可辩护"基于语义不确定性" |
| **API成本** | 固定2次图像调用 | 1次图像 + 条件TAC |
| **临床真实** | ⚠️ 模拟两位医生（可能都错） | ✅ 模拟一位医生的自我质疑 |
| **方法学严谨** | ⚠️ 触发条件较随意 | ✅ 量化不确定性，可调阈值 |

## 💡 关键创新点总结

### 1. **不确定性感知提示（Uncertainty-Aware Prompting）**
- 不是让模型只给答案
- 而是让模型**反思自己有多确定**
- 引入 `evidence_consistency` 和 `uncertainty_factors` 作为元认知指标

### 2. **语义不确定性量化（Semantic Uncertainty Quantification）**
- 不依赖随机采样（多次调用）
- 基于**语义指标**（证据矛盾、质量报告、混合倾向）
- 更稳定、更可解释

### 3. **条件性专家会诊（Conditional Expert Consultation）**
- TAC不是每例必用
- 只在模型"自己也不确定"时引入
- 符合临床：明确诊断不需要额外检查

### 4. **端到端可微分的可能性（未来扩展）**
- 当前规则量化可改为学习权重
- `uncertainty_score = MLP(confidence, cons, mixed, quality)`
- 可在验证集上学习最优阈值τ

## 🎯 一句话总结

> **"让模型先自我审视，只有它自己不确定时，才请TAC专家会诊。"**

这模拟了临床中**"医生皱眉→请专家"**的自然流程，而非**"随机抽两位医生，不一致就请会诊"**的机械流程。

## 附：阈值选择建议

| 场景 | 推荐τ | 理由 |
|------|-------|------|
| 强调高准确率 | 0.3 | 更敏感，宁可多触发 |

| 平衡（默认） | 0.4 | 折中方案 |

**建议**：先在验证集上测试τ=0.3, 0.4, 0.5，选择准确率与TAC使用率的最佳平衡点。
